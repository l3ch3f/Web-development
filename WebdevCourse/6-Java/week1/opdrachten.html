Opdracht 1 - Basic Syntax

Voordat je Java kan gebruiken moet je de syntax kunnen begrijpen. Doe onderzoek naar de volgende termen en leg uit wat ze (voor Java) betekenen:

  • Object: Een object word gemaakt vanuit een class.
  • Package: Groepeerd classes bij elkaar voor hun relatie met elkaar. Zie het als een folder in een file structuur.
  • Constructor: Een methode(zie het als een functie) om een object te initialiseren. De constructor wordt opgeroepen wanneer een object wordt gecreeerd van een class. Waarbij de attributes worden bepaald.
  • Instance: Zie het als een variabele declaren.
  • Method: Worden binnen een class declared, ze laten bepaalde acties uitvoeren.
  • Class: Een Class is de blauwprint van een object waar het object aan moet voldoen

Opdracht 2 - Variables

a) Waar wordt een local variable gedeclareerd?
Ze worden declared in methods, constructors of blocks.

b)Wat voor soort informatie sla je op in een instance variable?
Instance variables sla je op in een class, buiten methods, constructors of block. En het bevat waardes die moeten refereren door meer dan 1 method, constructor of block.

c) Wat voor soort informatie sla je op in een class variable?
Class variables ook wel static variables worden met de static keyword declared in een class, maar wel uit een method, constructor of block. Het komt maar 1 keer voor in een class, maar het kan wel vaker terug komen in een object wanneer het gecreeerd word.

d) Wat is het voordeel van het gebruiken van een class variable?
Het kan vaker opgeroepen worden bij het creeeren van nieuwe objecten.

Opdracht 3 - Acces Control Modifiers

  • Default: bezit geen keyqord, we declareren niet expliciet een acces modifier voor een class, field of method. Het is toegankelijk voor elke andere class in dezelfde package.
  • Private: Word alleen toegankelijk binnen de class zelf.
  • Public: Is toegankelijk vanuit elk andere class ongeacht de package.
  • Protected: Is alleen toegankelijk voor de subclasses in andere package of elke class in de package van de protected member class. Het kan niet op een class of interface gebruikt worden, wel op methods en fields.

Opdracht 4 - Non Acces Modifiers

Final word toegepast op Classes, methods en fields. Zie het als de const van javascript waarbij het maar 1 keer geinitialiseerd word en niet meer veranderlijk is.

Opdracht 5 - Strictly Typed

a) Wat is een voordeel van Strict Typing?
Je kan specifieker datatypes aangeven waarbij de datatype niet veranderd word.

b) Wat is een voordeel van Weak Typing?
Het kan voorkomen dat je niet veel rekening hoeft te houden met de dataType waarbij het programeren sneller gaat omdat je niet meer met Cascading te maken hebt.
